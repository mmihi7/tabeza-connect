## Root Cause

The core problem is that **chokidar (Node.js file watcher) cannot monitor `C:\Windows\System32\spool\PRINTERS`** due to Windows security restrictions. Even with Administrator access, the OS blocks directory enumeration (`scandir`) on that protected path — so chokidar never fires events, regardless of polling mode.

## The Real Fix: Use the Windows Print API Instead

Instead of watching the spooler directory, you need to monitor print jobs using the Windows Print API via a native Node.js approach. Here are two solid paths:

### Option 1: Use `win32-api` / PowerShell polling (quick fix)

Poll for print jobs using PowerShell's `Get-PrintJob` cmdlet from Node.js, which doesn't require spooler directory access:

```js
// printJobMonitor.js - replace spooler watcher entirely
const { exec } = require('child_process');

function pollPrintJobs(printerName, intervalMs = 1000) {
  let knownJobIds = new Set();

  setInterval(() => {
    const cmd = `powershell -Command "Get-PrintJob -PrinterName '${printerName}' | Select-Object -Property Id,Document,Size | ConvertTo-Json"`;
    exec(cmd, (err, stdout) => {
      if (err || !stdout.trim()) return;
      try {
        const jobs = [].concat(JSON.parse(stdout)); // handle single vs array
        jobs.forEach(job => {
          if (!knownJobIds.has(job.Id)) {
            knownJobIds.add(job.Id);
            console.log(`New print job detected: ${job.Document} (ID: ${job.Id}, Size: ${job.Size} bytes)`);
            // TODO: process job / enqueue receipt
          }
        });
      } catch (e) { /* ignore parse errors */ }
    });
  }, intervalMs);
}

pollPrintJobs('EPSON L3210 Series');
```

### Option 2: Use a Virtual Printer Port (most reliable, long-term)

Redirect print output to a folder instead of monitoring the locked spooler:

1. In Windows, create a new printer that uses a **"Local Port"** pointing to `C:\ProgramData\Tabeza\TabezaPrints`
2. Use the **"Generic / Text Only"** driver (already mentioned in your service instructions)
3. Your existing folder watcher (`C:\ProgramData\Tabeza\TabezaPrints`) will then catch every print job as a file — no permission issues at all

This is actually already configured in your service (`Watch Folder: C:\ProgramData\Tabeza\TabezaPrints`) — you just need to route the EPSON printer through a file port.

### Steps to set up the virtual port:
```
1. Open: Control Panel → Devices and Printers → Add a Printer
2. Choose: "Add a local printer or network printer with manual settings"
3. Port: "Create a new port" → Type: "Local Port"
4. Port name: C:\ProgramData\Tabeza\TabezaPrints\
5. Driver: Generic / Text Only
6. Printer name: TABEZA_RECEIPT (or whatever)
7. In your POS, print to this new printer
```

**My recommendation:** Go with Option 2 (virtual file port). It's the most stable, avoids all Windows security restrictions, and your watch folder monitoring code already works perfectly. The spooler approach is fundamentally blocked by OS security and not worth fighting.
