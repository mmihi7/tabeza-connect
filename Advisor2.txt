## 🔍 Root Cause

The Windows Print Spooler directory (`C:\Windows\System32\spool\PRINTERS`) has **extra security restrictions** that prevent even Administrators from:
- Listing directory contents (`scandir` returns `EPERM: operation not permitted`)
- Using file system watchers (chokidar fails silently)
- Even with `usePolling: true`, the directory access is blocked

## 🎯 Solution: Use Windows Print Job API Instead

Instead of monitoring spooler files directly, use the **Windows Print Job API** (`EnumJobs`, `OpenPrinter`) or **WMI** to monitor print jobs. This is the officially supported method and bypasses file system restrictions.

### Option 1: PowerShell + WMI Monitor (Recommended Quick Fix)

Create a PowerShell script that monitors print jobs via WMI and outputs to a folder TabezaConnect CAN access:

```powershell
# print-monitor.ps1
$printerName = "EPSON L3210 Series"
$outputPath = "C:\ProgramData\Tabeza\TabezaPrints"
$lastJobId = 0

while ($true) {
    try {
        $jobs = Get-CimInstance -ClassName Win32_PrintJob | Where-Object { $_.Name -like "*$printerName*" }
        
        foreach ($job in $jobs) {
            if ($job.JobId -gt $lastJobId) {
                $lastJobId = $job.JobId
                
                # Create receipt data
                $receipt = @{
                    timestamp = Get-Date -Format "yyyy-MM-ddTHH:mm:ss"
                    printer = $printerName
                    jobId = $job.JobId
                    document = $job.Document
                    pages = $job.TotalPages
                    status = $job.JobStatus
                    data = "Print job captured via WMI"
                } | ConvertTo-Json
                
                $filename = "receipt-$(Get-Date -Format 'yyyyMMdd-HHmmss')-$($job.JobId).json"
                $receipt | Out-File -FilePath "$outputPath\$filename" -Encoding UTF8
                
                Write-Host "Captured job $($job.JobId): $($job.Document)"
            }
        }
    } catch {
        Write-Error "Error: $_"
    }
    Start-Sleep -Milliseconds 500
}
```

**Run this as Administrator** alongside your TabezaConnect service.

### Option 2: Node.js Native Print Job Monitor

Replace your `spoolMonitor.js` with this Windows API-based approach using `node-ffi` or a PowerShell wrapper:

```javascript
// printJobMonitor.js
const { exec } = require('child_process');
const fs = require('fs');
const path = require('path');

class PrintJobMonitor {
  constructor(printerName, outputPath) {
    this.printerName = printerName;
    this.outputPath = outputPath;
    this.knownJobs = new Set();
    this.isRunning = false;
  }

  async start() {
    console.log(`🔍 Monitoring print jobs for: ${this.printerName}`);
    this.isRunning = true;
    this.monitorLoop();
  }

  async monitorLoop() {
    while (this.isRunning) {
      try {
        const jobs = await this.getPrintJobs();
        
        for (const job of jobs) {
          if (!this.knownJobs.has(job.JobId)) {
            this.knownJobs.add(job.JobId);
            await this.processJob(job);
          }
        }
        
        // Cleanup old job IDs (keep last 100)
        if (this.knownJobs.size > 100) {
          const toDelete = Array.from(this.knownJobs).slice(0, 50);
          toDelete.forEach(id => this.knownJobs.delete(id));
        }
        
      } catch (error) {
        console.error('❌ Monitor error:', error.message);
      }
      
      await this.sleep(1000); // Poll every second
    }
  }

  getPrintJobs() {
    return new Promise((resolve, reject) => {
      const psCommand = `
        Get-CimInstance -ClassName Win32_PrintJob | 
        Where-Object { $_.Name -like "*${this.printerName}*" } |
        Select-Object JobId, Document, TotalPages, JobStatus, Name |
        ConvertTo-Json -Compress
      `;
      
      exec(`powershell -Command "${psCommand}"`, (error, stdout) => {
        if (error) {
          reject(error);
          return;
        }
        
        try {
          const data = JSON.parse(stdout);
          resolve(Array.isArray(data) ? data : [data]);
        } catch (e) {
          resolve([]);
        }
      });
    });
  }

  async processJob(job) {
    console.log(`📄 New print job detected: ${job.Document} (ID: ${job.JobId})`);
    
    const receipt = {
      timestamp: new Date().toISOString(),
      printer: this.printerName,
      jobId: job.JobId,
      document: job.Document,
      pages: job.TotalPages,
      status: job.JobStatus,
      source: 'wmi-print-monitor',
      data: `Print job: ${job.Document} | Pages: ${job.TotalPages}`
    };

    const filename = `receipt-${Date.now()}-${job.JobId}.json`;
    const filepath = path.join(this.outputPath, filename);
    
    try {
      fs.writeFileSync(filepath, JSON.stringify(receipt, null, 2));
      console.log(`✅ Receipt saved: ${filename}`);
    } catch (error) {
      console.error(`❌ Failed to save receipt: ${error.message}`);
    }
  }

  sleep(ms) {
    return new Promise(resolve => setTimeout(resolve, ms));
  }

  stop() {
    this.isRunning = false;
  }
}

// Usage
const monitor = new PrintJobMonitor(
  'EPSON L3210 Series', 
  'C:\\ProgramData\\Tabeza\\TabezaPrints'
);

monitor.start();
```

### Option 3: Change Spooler Directory (If You Must Use File Monitoring)

If you absolutely need to monitor spool files, you can change the spool directory to a location with less restrictions:

```powershell
# Run as Administrator
# Change spool directory for specific printer
$printerName = "EPSON L3210 Series"
$newSpoolPath = "C:\TabezaSpool"

# Create directory with full permissions
New-Item -ItemType Directory -Force -Path $newSpoolPath
$acl = Get-Acl $newSpoolPath

# Add Everyone with Full Control
$rule = New-Object System.Security.AccessControl.FileSystemAccessRule(
    "Everyone", "FullControl", "ContainerInherit,ObjectInherit", "None", "Allow"
)
$acl.SetAccessRule($rule)
Set-Acl $newSpoolPath $acl

# Update printer spool directory in registry
$regPath = "HKLM:\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Print\Printers\$printerName"
Set-ItemProperty -Path $regPath -Name "SpoolDirectory" -Value $newSpoolPath

# Restart spooler
Restart-Service -Name Spooler -Force
```

Then update TabezaConnect to watch `C:\TabezaSpool` instead.

## 💡 Why This Happens

According to Microsoft documentation , Windows Print Spooler runs as `SYSTEM` and has strict security on its spool directory to prevent:
- Privilege escalation attacks (CVE-2020-1048, CVE-2022-21999) 
- Malicious file manipulation
- Unauthorized access to print data

The `EPERM` error you saw confirms this security boundary is active. Using the **Print Job API** (via WMI) is the correct, supported approach that respects Windows security while still allowing you to capture receipt data.
