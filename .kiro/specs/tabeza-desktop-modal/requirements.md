# Requirements Document: POS Receipt Assignment Modal

## Introduction

The POS Receipt Assignment Modal is a browser-based interface integrated into the Tabeza Staff PWA that enables waiters to assign POS receipts to customer tabs. This feature eliminates the manual paper delivery workflow in Tabeza Basic mode by providing an immediate modal interface when receipts are printed from the POS system.

The system works in conjunction with the existing Tabeza Printer Service (a minimal Windows service that intercepts print jobs and sends them to the cloud). When a receipt is printed, the cloud broadcasts a real-time event to the Staff PWA, which displays a modal overlay for tab assignment. This approach keeps the waiter in their primary work environment (the browser) without requiring context switching to a separate application.

## Glossary

- **POS**: Point of Sale system - the existing cash register/ordering system used by the venue
- **Captain's Order**: A printed receipt generated by the POS system showing order details
- **Tab**: A customer's open transaction context in Tabeza containing their orders and payments
- **Modal**: A dialog overlay that appears within the browser requiring user interaction
- **Staff PWA**: The Tabeza Staff Progressive Web App running in the browser
- **Printer Service**: The existing Tabeza Printer Service (Windows service) that intercepts print jobs
- **Real-Time Event**: Server-Sent Events (SSE) or WebSocket message pushed from cloud to browser
- **Digital Receipt**: An electronic version of the captain's order delivered to the customer's mobile device
- **Tabeza Basic Mode**: Operating mode where POS is the digital authority and Tabeza mirrors receipts
- **Waiter**: Staff member who takes orders and operates the POS system
- **Tab Assignment**: The action of linking a POS receipt to a specific customer tab
- **Unmatched Receipt**: A receipt from the POS that has not yet been assigned to a customer tab

## Requirements

### Requirement 1: Real-Time Receipt Event Reception

**User Story:** As a waiter using the Staff PWA, I want to receive immediate notification when a receipt is printed from the POS, so that I can assign it to a customer tab right away.

#### Acceptance Criteria

1. WHEN the Staff PWA is loaded, THE Application SHALL establish a persistent Server-Sent Events (SSE) connection to the cloud API
2. WHEN a receipt is intercepted by the Printer Service and sent to the cloud, THE Cloud_API SHALL broadcast a real-time event to all connected Staff PWA clients for that venue
3. WHEN the Staff PWA receives a receipt event, THE Application SHALL trigger the modal display within 500ms
4. IF the SSE connection is lost, THEN THE Staff_PWA SHALL automatically attempt reconnection every 5 seconds
5. THE Staff_PWA SHALL maintain the SSE connection as long as the browser tab remains open

### Requirement 2: Modal Overlay Display

**User Story:** As a waiter who just printed from POS, I want a modal to appear immediately in my browser showing the receipt, so that I can assign it to a customer tab without leaving the Staff PWA.

#### Acceptance Criteria

1. WHEN a receipt event is received, THE Staff_PWA SHALL display a modal overlay within 500ms
2. THE Modal SHALL appear centered on the screen with a semi-transparent backdrop
3. THE Modal SHALL display the parsed receipt content in a readable format with line items, quantities, and prices
4. THE Modal SHALL remain visible until the waiter completes tab assignment or dismisses it
5. WHEN the modal is displayed, THE Staff_PWA SHALL play a subtle notification sound to alert the waiter

### Requirement 3: Tab Selection Interface

**User Story:** As a waiter assigning an order, I want to see a list of all open customer tabs with table numbers and names, so that I can quickly select the correct customer.

#### Acceptance Criteria

1. WHEN the modal is displayed, THE Staff_PWA SHALL fetch all open tabs for the current venue from the local cache or API
2. THE Modal SHALL display a searchable list of open tabs showing tab number, table number (if available), and customer identifier
3. WHEN the waiter types in the search field, THE Modal SHALL filter the tab list in real-time by tab number or table number
4. THE Modal SHALL display tabs in descending order by creation time (most recent first)
5. WHEN no open tabs exist, THE Modal SHALL display a message "No open tabs available" and disable the assignment button

### Requirement 4: One-Click Tab Assignment

**User Story:** As a waiter in a hurry, I want to assign the receipt with one click, so that I can get back to serving customers quickly.

#### Acceptance Criteria

1. WHEN a waiter selects a tab from the list, THE Modal SHALL enable the "Send to Customer" button
2. WHEN the waiter clicks "Send to Customer", THE Staff_PWA SHALL send the receipt ID and tab ID to Tabeza Cloud API
3. WHEN the API confirms successful assignment, THE Staff_PWA SHALL display a success confirmation for 2 seconds
4. WHEN the API returns an error, THE Staff_PWA SHALL display the error message and allow retry
5. THE Modal SHALL close automatically after successful assignment and confirmation display

### Requirement 5: Background Tab Notification

**User Story:** As a waiter who might have multiple browser tabs open, I want to be notified when a receipt arrives even if the Staff PWA tab is not active, so that I don't miss assignments.

#### Acceptance Criteria

1. WHEN a receipt event is received and the Staff PWA tab is not active, THE Application SHALL display a browser notification
2. THE Browser_Notification SHALL show the receipt total and a preview of items
3. WHEN the waiter clicks the browser notification, THE Browser SHALL focus the Staff PWA tab and display the modal
4. IF browser notifications are not permitted, THEN THE Staff_PWA SHALL display an in-app banner requesting notification permission
5. THE Staff_PWA SHALL respect the user's notification preferences and allow them to be toggled in settings

### Requirement 6: Offline Capability and Caching

**User Story:** As a waiter during network issues, I want the app to cache tab information, so that I can still assign orders when internet is slow.

#### Acceptance Criteria

1. WHEN the Staff_PWA successfully fetches open tabs, THE Application SHALL cache the tab list in browser storage with a timestamp
2. WHEN network connectivity is lost, THE Staff_PWA SHALL use the cached tab list and display a "Using cached data" indicator
3. WHEN the waiter attempts assignment while offline, THE Staff_PWA SHALL queue the assignment request in IndexedDB for retry
4. WHEN network connectivity is restored, THE Staff_PWA SHALL automatically retry queued assignments in order
5. THE Staff_PWA SHALL refresh the cached tab list every 30 seconds when online

### Requirement 7: Visual Confirmation and Feedback

**User Story:** As a waiter, I want to see a confirmation when the receipt is sent, so that I know the customer received it.

#### Acceptance Criteria

1. WHEN receipt assignment is successful, THE Modal SHALL display a green checkmark icon with "Receipt sent to Tab #[number]" message
2. THE Success_Confirmation SHALL remain visible for 2 seconds before the modal closes automatically
3. WHEN assignment fails, THE Modal SHALL display a red error icon with the specific error message
4. WHEN assignment is in progress, THE Modal SHALL display a loading spinner and disable all buttons
5. THE Modal SHALL provide a "Cancel" button that allows the waiter to dismiss without assigning

### Requirement 8: Unmatched Receipts Management

**User Story:** As a waiter, I want to access receipts I missed or dismissed, so that I can assign them later if needed.

#### Acceptance Criteria

1. WHEN a receipt is dismissed or ignored, THE Staff_PWA SHALL store it in the "Unmatched Receipts" list
2. THE Staff_PWA SHALL provide an "Unmatched Receipts" section accessible from the main navigation
3. WHEN viewing unmatched receipts, THE Staff_PWA SHALL display all receipts from the past hour sorted by timestamp
4. WHEN a waiter clicks an unmatched receipt, THE Staff_PWA SHALL display the assignment modal for that receipt
5. THE Staff_PWA SHALL automatically remove receipts older than 1 hour from the unmatched list

### Requirement 9: Connection Status Indication

**User Story:** As a waiter, I want to see the connection status to the receipt service, so that I know if the system is working properly.

#### Acceptance Criteria

1. WHEN the SSE connection is active, THE Staff_PWA SHALL display a green "Connected" indicator in the header
2. WHEN the SSE connection is lost, THE Staff_PWA SHALL display a yellow "Reconnecting..." indicator
3. WHEN reconnection fails after 3 attempts, THE Staff_PWA SHALL display a red "Disconnected" indicator with a manual reconnect button
4. WHEN the waiter clicks the manual reconnect button, THE Staff_PWA SHALL attempt to reestablish the SSE connection
5. THE Staff_PWA SHALL log connection events to the browser console for debugging purposes

### Requirement 10: Error Handling and Recovery

**User Story:** As a waiter, I want the app to handle errors gracefully, so that I can continue working even when issues occur.

#### Acceptance Criteria

1. WHEN the SSE connection fails, THE Staff_PWA SHALL display a notification and attempt reconnection every 5 seconds
2. WHEN the Tabeza Cloud API is unreachable, THE Staff_PWA SHALL queue assignments in IndexedDB and display "Working offline" status
3. WHEN receipt data is incomplete or malformed, THE Staff_PWA SHALL display the available data and allow manual tab assignment
4. WHEN an unhandled error occurs, THE Staff_PWA SHALL log the error to the console and display a user-friendly error message
5. THE Staff_PWA SHALL provide a "Report Issue" button that captures error logs and allows the waiter to submit feedback

### Requirement 11: Receipt Display Formatting

**User Story:** As a waiter, I want to see receipts formatted clearly in the modal, so that I can verify the order details before assigning.

#### Acceptance Criteria

1. WHEN a receipt is displayed, THE Modal SHALL show: venue name, timestamp, order items, quantities, prices, subtotal, tax, and total
2. THE Modal SHALL display receipt data in a structured format with clear sections for header, line items, and totals
3. WHEN line items exceed 10 items, THE Modal SHALL display a scrollable list with the total always visible
4. THE Modal SHALL handle special characters and non-ASCII text correctly
5. WHEN receipt data is incomplete, THE Modal SHALL display available fields and mark missing fields as "N/A"

### Requirement 12: Multi-Venue Support

**User Story:** As a staff member who works at multiple venues, I want the modal to only show tabs for the venue I'm currently managing, so that I don't accidentally assign to the wrong venue.

#### Acceptance Criteria

1. WHEN the Staff_PWA is loaded, THE Application SHALL determine the current venue from the user's session
2. THE Staff_PWA SHALL only display tabs for the current venue when showing the tab selection list
3. WHEN receipt events are received, THE Staff_PWA SHALL only process receipts for the current venue
4. THE Staff_PWA SHALL display the current venue name prominently in the modal header
5. WHEN a staff member switches venues in the PWA, THE Staff_PWA SHALL reestablish the SSE connection for the new venue

### Requirement 13: Performance and Responsiveness

**User Story:** As a waiter, I want the modal to appear and respond quickly, so that I can assign receipts without delays during busy service.

#### Acceptance Criteria

1. THE Staff_PWA SHALL display the modal within 500ms of receiving a receipt event
2. THE Staff_PWA SHALL render the tab selection list within 300ms of modal display
3. WHEN the waiter types in the search field, THE Staff_PWA SHALL filter results within 100ms
4. THE Staff_PWA SHALL complete the assignment API call within 3 seconds under normal network conditions
5. THE Staff_PWA SHALL maintain smooth scrolling and interactions even with 100+ open tabs

### Requirement 14: Security and Data Protection

**User Story:** As a venue owner, I want customer and order data to be secure, so that we comply with data protection requirements.

#### Acceptance Criteria

1. THE Staff_PWA SHALL transmit all data to Tabeza Cloud API over HTTPS only
2. THE Staff_PWA SHALL not store receipt content in browser storage after successful assignment
3. WHEN logging errors to the console, THE Staff_PWA SHALL redact sensitive information including customer names and payment details
4. THE Staff_PWA SHALL validate SSL certificates when connecting to Tabeza Cloud API
5. THE Staff_PWA SHALL clear cached receipt data when the user logs out

### Requirement 15: Browser Compatibility

**User Story:** As a venue manager, I want the modal to work on different devices and browsers, so that staff can use their preferred devices.

#### Acceptance Criteria

1. THE Staff_PWA SHALL support Chrome, Firefox, Safari, and Edge browsers (latest 2 versions)
2. THE Staff_PWA SHALL work on desktop, tablet, and mobile screen sizes with responsive layout
3. WHEN accessed on a mobile device, THE Modal SHALL adapt to touch interactions and smaller screens
4. THE Staff_PWA SHALL gracefully degrade features if browser APIs (like notifications) are not available
5. THE Staff_PWA SHALL display a warning if the browser version is unsupported
